# Teacher Guide: Lesson 01 — Hello World

> **Prep time: 10 minutes.** Read this before class. You'll be ready.

---

## What This Lesson Teaches

Students create their first HTML file and open it in a browser. They learn that web pages
are just text files with special tags. By the end, every student should have a working
web page with a heading, paragraphs, a list, and bold/italic text.

## What Students Will Build

A simple "About Me" page with:
- A heading with their name or nickname (NOT full name)
- A few paragraphs about their interests
- A bulleted list
- Bold and italic text

**Expected result:** A plain-looking but functional web page. It won't look fancy —
that's on purpose. Skill-02 adds styling.

## Before Class

- [ ] Computers are on and logged in
- [ ] Text editor is installed and opens correctly
- [ ] Students know where to save files (Desktop? Documents? A class folder?)
- [ ] You've opened the example file `examples/html-basics/index.html` to see the finished product

## Timing (45-minute class)

| Time | Activity |
|---|---|
| 0:00–0:05 | Introduction: "Today you become web developers" |
| 0:05–0:10 | Demo: Open any website, right-click, "View Source" — it's all text! |
| 0:10–0:35 | Students work through SKILL-01 (Steps 1–5) |

## The Demo That Hooks Them

Open any simple website in a browser. Right-click and select "View Page Source."
Point out that everything they see — every heading, every paragraph, every image —
is defined by text tags. Then say: "By the end of today, you'll know how to write
these tags yourself."

This is the "whoa" moment. Don't rush it.

## Common Questions

**"What does DOCTYPE mean?"**
"It tells the browser 'this is a modern HTML page.' Think of it as the browser's
instruction manual — it says 'read this the modern way.' You'll always put it at the
top of every page."

**"Why do tags come in pairs?"**
"The opening tag says 'start doing this thing' and the closing tag says 'stop.'
Like parentheses in math: you need both. `<p>` says 'start a paragraph' and `</p>`
says 'end the paragraph.'"

**"My page is blank!"**
Check these in order: (1) Is the file saved? (2) Does it end in .html? (3) Is the
file open in a browser (not the text editor)? (4) Are there typos in the tag names?

**"Can I add pictures?"**
"Great question! We'll learn that soon. For today, let's focus on text and structure."

**"Can I change the colors?"**
"That's exactly what we're doing in the NEXT lesson! For today, we're building the
skeleton. Next time we add the paint."

## Common Mistakes

| Mistake | What You'll See | Fix |
|---|---|---|
| File saved as .txt not .html | Browser shows raw code | Rename the file to .html |
| Missing closing tag | Page looks weird/broken | Find the tag without a `/` partner |
| Typo in tag name | Nothing happens | Check spelling: `<h1>` not `<h!>` |
| Opening file in text editor | Sees code, not a page | Right-click → Open With → Browser |

## What "Success" Looks Like

Every student should leave class with:
- A working .html file they created themselves
- An understanding that web pages are text files with tags
- At least a heading and a paragraph on their page
- Excitement to make it look better (that's Lesson 02)

**It's okay if:**
- Some students only finish Steps 1–3
- The code has imperfect indentation
- Students choose silly (but appropriate) content for their pages

**It's NOT okay if:**
- Students use real full names, addresses, or personal info
- Students are frustrated and haven't gotten help
- Anyone copies another student's file instead of typing their own

## Safety Notes for This Lesson

- Remind students: first names or nicknames only, no personal info
- Walk the room and glance at content — redirect anything inappropriate
- If AI is enabled, it's not needed for this lesson. Students should type their own code.

---

*After this lesson, students understand HTML structure. They can create a file, add
tags, and see the result in a browser. That's a real accomplishment — make sure they
know it.*
