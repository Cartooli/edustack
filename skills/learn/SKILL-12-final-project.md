# SKILL-12: Final Project — Show What You Can Do

## Overview
This is it — your capstone project. You'll plan, design, build, and present a complete
web application. You choose what to build. You use every skill you've learned.
This project is yours.

## What You Need
- Everything from Skills 01–11
- 2–3 class periods to build
- Your creativity and determination

## Key Idea
Real developers don't build what someone tells them to build — they identify a problem
and create a solution. Your final project starts with a question: **What problem do you
want to solve, or what experience do you want to create?**

## Phase 2: Build (Day 1 remaining + Day 2)

### Build Order

Follow this order. Don't skip ahead!

**1. HTML structure (30 min)**
- Build every section and element
- Don't worry about looks
- Use real content (not "lorem ipsum")
- Test: Everything shows up in the browser

**2. CSS styling (30 min)**
- Apply your color scheme
- Layout the page
- Style interactive elements (buttons, inputs)
- Add at least one animation
- Test: It looks good and is easy to read

**3. JavaScript functionality (45 min)**
- Start with the most important feature
- Get it working before adding the next
- Test each feature as you build it
- Use `console.log()` to verify data is correct

**4. Polish (15 min)**
- Check for bugs (open console, click everything)
- Clean up your code (add comments, fix indentation)
- Run the student checklist: `skills/review/STUDENT-CHECKLIST.md`

### Rules While Building

- **Build one feature at a time.** Don't try to do everything at once.
- **Test constantly.** Save, refresh, check after every change.
- **Ask for help when stuck.** Use your AI tutor, ask a classmate, ask your teacher.
- **It's okay to simplify.** If a feature is too hard, simplify it. A working simple
  feature is better than a broken complex one.
- **Save your work frequently.** Save after every successful change.

## Phase 3: Test (Day 2 or Day 3, first 15 minutes)

Before presenting, test everything:

### Functionality Test
- [ ] Every button does what it should
- [ ] Every form validates input
- [ ] Edge cases work (empty input, first item, last item)
- [ ] No errors in the browser console

### Design Test
- [ ] Page is readable and organized
- [ ] Colors have good contrast
- [ ] Interactive elements look clickable
- [ ] Animations are smooth

### Safety Test
- [ ] No personal information in the code
- [ ] No external links
- [ ] User input is validated and sanitized
- [ ] Content is school-appropriate

## Phase 4: Present (Day 3)

You'll have 3–5 minutes to present your project. Here's a structure:

1. **What I built** (30 seconds)
   "I built a [name], which [what it does]."

2. **Demo** (2 minutes)
   Show it working. Click through the features. Show one thing you're proud of.

3. **What I learned** (1 minute)
   What was the hardest part? What would you do differently? What surprised you?

4. **Questions** (1 minute)
   Classmates can ask you about your project.

### Presentation Tips
- Practice your demo once before presenting
- Know which features work perfectly (show those)
- It's okay if something isn't perfect — talk about what you learned from the struggle
- Be proud of what you built! You wrote this code. That's amazing.
