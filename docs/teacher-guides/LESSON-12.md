# Teacher Guide: Lesson 12 — Final Project

> **Prep time: 20 minutes.** This is a 3-day capstone. Plan the full arc.

---

## What This Lesson Teaches

Students design, build, and present a web project of their choice. This is the
capstone of the entire curriculum — students demonstrate everything they've learned
across all 11 previous lessons.

## What Students Will Build

A project of their choice. Suggested categories:
- **Personal portfolio** — A multi-page showcase of their best work
- **Interactive tool** — Calculator, converter, quiz, or timer
- **Information display** — Dashboard, catalog, or reference guide
- **Creative project** — Story page, art gallery, or interactive poem

**Expected result:** A complete, polished, working web project that the student
designed and built themselves.

## Before Class (Day 1)

- [ ] Computers are on and logged in
- [ ] Blank paper and pencils for planning
- [ ] Print the project brief template from SKILL-12
- [ ] Decide: How will students present? (Screen share, gallery walk, projector)
- [ ] Plan for 3 class periods: Day 1 (plan), Day 2 (build), Day 3 (test + present)
- [ ] Prepare celebration: certificates, applause protocol, or class showcase

## Timing (3-day lesson)

### Day 1: Plan (45 minutes)

| Time | Activity |
|---|---|
| 0:00–0:05 | Introduction: "This is YOUR project. You decide what to build." |
| 0:05–0:10 | Show project categories and examples |
| 0:10–0:25 | Students write project briefs (what, why, how, features) |
| 0:25–0:35 | **Teacher reviews briefs** — check scope and safety (see notes below) |
| 0:35–0:45 | Students sketch wireframes on paper |

### Day 2: Build (45 minutes)

| Time | Activity |
|---|---|
| 0:00–0:05 | Check-in: "What's your plan for today?" |
| 0:05–0:40 | Build time — teacher circulates, helps debug, encourages |
| 0:40–0:45 | "Save your work! Tomorrow we test and present." |

### Day 3: Test and Present (45 minutes)

| Time | Activity |
|---|---|
| 0:00–0:05 | Final build time and testing |
| 0:05–0:10 | Students test each other's projects and give feedback |
| 0:10–0:35 | Presentations or gallery walk |
| 0:35–0:42 | Class vote: "Most Creative," "Best Design," "Coolest Feature" |
| 0:42–0:48 | Reflection: "What did you learn? What would you build next?" |
| 0:48–0:50 | Celebration: "You are web developers. For real." |

## The Demo That Hooks Them

Don't demo a specific project — instead, show a quick montage of what's possible:
a styled portfolio, an interactive quiz, a data dashboard. Then say: "You know
how to build ALL of these. Now pick the one that excites YOU."

## Common Questions

**"I don't know what to build."**
"What's something you wish existed? A tool you'd use? A page you'd show someone?
Start there. If you're really stuck, build a portfolio showcasing the projects
you've already made."

**"Can I work with a partner?"**
"If your teacher allows it, yes — but both partners must write code and be able to
explain every part. No one rides for free."

**"My project is too big."**
"What's the smallest version that still works? Build that first. You can always
add features, but a working simple app beats a broken complex one."

**"Can I use something we didn't learn?"**
"Stick with HTML, CSS, and JavaScript — the things you know well. The goal is to
show mastery, not to learn something new under pressure."

## Common Mistakes

| Mistake | What You'll See | Fix |
|---|---|---|
| Scope too big | Student overwhelmed by Day 2 | "Cut 2 features. What's the core?" |
| No plan | Coding without direction | Send them back to paper: "Sketch first." |
| All style, no function | Pretty page that does nothing | "What happens when I click something?" |
| All function, no style | Working app that looks rough | "Spend 10 minutes on CSS. It matters." |

## Safety Notes for This Lesson

- **Review all project briefs on Day 1** for inappropriate content, themes, or data
- First names and nicknames only — no personal details in projects
- Monitor content throughout build time — students have more creative freedom here
- All projects must use `textContent` for any user input display
- No external URLs, images, or resources — everything self-contained
- If AI is enabled, students may use it as a helper, but the project must be their own
- **Portfolio-based grading recommended:** Grade the project holistically based on
  effort, creativity, completeness, and code quality — not perfection
- Celebrate EVERY student. Some projects will be simple. That's okay.
  A student who builds a working page from scratch deserves applause.

*This is the final lesson. Students leave with a project they're proud of and the
