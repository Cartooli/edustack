Welcome to EduStack!

This guide will help you set up EduStack in your classroom. The entire setup takes about 15 minutes and only needs to be done once.

You don't need to know how to code. EduStack is designed so that you learn each lesson 15 minutes before you teach it.

What You Need

  • A computer with Git installed (most school IT departments can do this)
  • A text editor (VS Code recommended — it's free)
  • A web browser (Chrome, Firefox, Edge, or Safari)
  • 15 minutes to set up before your first class

Setup (One Time)

1Clone EduStack

Open a terminal (or command prompt) and run:

git clone https://github.com/Cartooli/edustack.git

This downloads EduStack to your computer. It's just a folder with files — no installation needed.

2Enter the Directory

cd edustack

This moves you into the EduStack folder.

3Run the Setup Script

./setup.sh

This validates your environment. It doesn't install anything — just checks that everything is ready.

That's it! No npm install. No API keys. No accounts. No admin privileges needed.

Your First Class (30 minutes)

Before Class Checklist:

Read the teacher guide for Lesson 1 (5 min): docs/teacher-guides/LESSON-01.md
Open the example file in a browser to see what students will build: examples/html-basics/index.html
(Optional) Try the first 3 steps of Skill-01 yourself

During Class:

  1. Open the curriculum hub: Open index.html in your browser
  2. Select today's lesson: Use the "Today's Lesson" quick-start to open both the teacher guide and student skill
  3. Students follow the steps: Each skill is a guided, step-by-step activity
  4. Review together: Students share what they built on their screens

Getting Updates

To get the latest lessons and safety patches:

cd edustack
git pull origin main

EduStack follows semantic versioning. Safety patches are released immediately. New lessons are released on a predictable schedule.

AI Features (Optional)

EduStack can optionally integrate with Claude (via Claude Code or the API) to provide students with an AI coding tutor. This is entirely optional — all lessons work without AI.

If AI features will be used:

Claude Code or API access configured by IT department
Review docs/SAFETY.md and be comfortable with the safety system
Set config/teacher-settings.json to "ai_enabled": true

Remember: You are still the teacher. The AI is a tool, not a replacement. You can disable AI at any time if needed.

Troubleshooting

  • "The page is blank" → Check if the file was saved. Check for a typo in the filename.
  • "The colors don't show" → Check for a missing semicolon in CSS. Most common beginner error.
  • "Nothing happens when I click" → Check the browser console (F12) for JavaScript errors.
  • "The AI isn't responding" → Check if AI features are enabled in config/teacher-settings.json.
  • "I can't git pull" → Ask IT to check network access to GitHub.

Getting Help

  • Technical issues: Open an issue on the EduStack GitHub repository
  • Pedagogical questions: Check the teacher guides or open a discussion
  • Safety concerns: Follow the incident response process in docs/SAFETY.md

Next Steps

You're ready to start teaching! Here's what to do next:

  1. Open the curriculum hub to browse all lessons
  2. Read the teacher guide for Lesson 1
  3. Try the first few steps yourself (optional but recommended)
  4. Plan your first class using the timing guide in the teacher guide

You've got this! You're already doing something amazing by bringing coding into your classroom. EduStack is here to make that easier.
